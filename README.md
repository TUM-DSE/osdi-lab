## Course Overview

In this practical course, participants will gain a thorough understanding of the challenges and issues in designing and implementing modern multi-core operating systems. They will apply and extend their knowledge in systems, software engineering, project management, and teamwork.

During the course, the students will work together in small groups to build a fairly complete operating system. The course covers the design and implementation of various operating system concepts, including, but not limited to: memory management, scheduling, inter-process communication, inter-core synchronization, protection and access control, device drivers, file systems, and networking.

Moreover, the course pays particular attention to designing system software architectures that differ from traditional monolithic architectures in Unix/Linux and Windows.

## Objectives

- Teach general operating systems principles, using a real research operating system to illustrate
  them and by reading the research papers which propose some of the ideas that the particular OS builds on.
- Give a broader perspective on operating systems which do not look like Linux, Unix, or Windows.
- Provide exposure to the practical experience of working on OS code on real "metal", including
  debugging, hardware access, etc. This kind of experience is hard to gain merely from reading books
  or papers.
- Introduce a sense of the complexity of a real OS, rather than simplified teaching OSes often used
  in more basic courses.
